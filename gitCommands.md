
### Ветвления

`git branch` — список всех веток
`git checkout -b name` — создать ветку и перейти на неё
`git branch -d name` — удалить ветку
`git branch -D name` — удалить ветку с коммитами (принудительно)
`git branch -a` — показать локальные и удаленные ветки

### Работа с коммитами

###### Режимы git reset (смещение HEAD на указанный коммит)

*  `--soft`— Перемещает только указатель ветки. Изменения сохраняются на диске и в индексе (готовы к коммиту, в `git status` они зеленые). Безопасно.
*  `--mixed`(по умолчанию) — Перемещает указатель ветки и сбрасывает индекс. Изменения сохраняются на диске, но из индекса удаляются (в `git status` они красные). Безопасно.
*  `--hard` — Полностью стирает историю и изменения. Синхронизирует файлы на диске и индекс с указанным коммитом. Уничтожает незакоммиченную работу. **Опасно! Перед применением советуем создать бэкапы файлов**
