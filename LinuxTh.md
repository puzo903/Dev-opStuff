### Регулярные выражения:
 sudo apt-get install

#### Позиция в строке (якоря)
`^` — начало строки. Выражение ^Linux найдет строки, которые начинаются со слова Linux.  
`$` — конец строки. Выражение done$ найдет строки, которые заканчиваются на done.  
`^$` — пустая строка (символ начала сразу переходит в символ конца).  

#### Поиск символов
`.`(точка) — один любой символ (буква, цифра, пробел).  
`[abc]` — любой один символ из указанных в скобках (или a, или b, или c).  
`[a-z]` — любая строчная буква от a до z (диапазон).  
`[0-9]` — любая одна цифра.  
`[^0-9]` — любой символ, кроме цифры (знак ^ внутри квадратных скобок
означает отрицание).  

### Количество повторений (квантификаторы)
`*` — ноль или более повторений предыдущего символа.  
`+` — одно или более повторений (доступно в ERE).  
`?` — ноль или одно повторение (предыдущий символ либо есть, либо его нет).  
`{n}` — ровно n повторений символа.  

### Стандартные команды:

`uptime` — показывает как долго система работает;

`echo $PATH` — показывает пути к директориям из которых linux берет команды (при желании можно создавать новые директории и авторские команды);  
`echo "*text*"` — вывод текста в терминал (кавычки опцианальны);  
`sudo` — super user do, выполнение от имени администратора;  
`wget *webadress*` — скачать файл по адресу;  
`sudo apt-get install *packname*` — скачать пакет;  
`sudo apt-get remove *packname*` — удалить пакет;  
`cat /etc/apt/sources.list` — лист репозиториев, откуда Linux берёт пакеты;  
sudo dpkg -i *packname*.deb — распаковать пакет .deb;  
sudo dpkg -r *packname*.deb — удалить пакет .deb;

Флаги `-d` и `-f` работают в паре:
- **`-d` (delimiter / разделитель):** Указывает символ, который разделяет колонки в тексте (по умолчанию это знак табуляции).

`cp *filename* *dirpath*` — копирование файла. В конце можно указать имя копии;

`rm` — удалить файл;  
`mv` — переместить файл или директорию или изменить имя;  
`mkdir` (*exsistdir*/*newdir*)(-p *newdir1*/*newdir2*) — создать директорию;  
`rmdir` — стереть пустую директорию;  
`rm -R *dirname*` — рекурсивно стирает всё содержимое указанной директории;  
`cp -R *outdir* *indir2*` — копирование всей директории из одной в другую;  
`sudo rm -R / --no-preserve-root` — **даблэй волки хотят падвигаца**;  
`ln -s `— создать линк на файл/директорию;  
`ln` — создать точную связанную с оригиналом копию;  
`ln -s` — создать линк на файл/директорию;  
`ln *originame* *copyname*` — создать точную связанную с оригиналом копию;

### Процессы:

`top` — отображает все действующие процессы и загруженность системы в реальном времени;  
`free` — показывает состояние памяти в байтах;  
`free -h` — показывает состояние памяти в Mb, Gb;  
`ps` — показать снимок запущенных из терминала процессов в реальном времени;  
`ps aux` — показывает **полный список всех запущенных в системе процессов** в реальном времени.
- **`a`** — показать процессы **всех пользователей** (а не только ваши собственные).
    
- **`u`** — вывести подробную информацию в **ориентированном на пользователя** формате (показывает имя владельца, загрузку CPU, памяти и т.д.).
    
- **`x`** — показать процессы, **не привязанные к терминалу** (фоновые службы, демоны и системные процессы, которые работают сами по себе);

`dmesg` — просмотр логов ядра системы (часто используется вместе с | grep);  


### Пользователи и группы:

`cat etc/passwd` — отображение всех пользователей системы, их пароля, номера, группы, комментария, домашней директории и shell по умолчанию;  
`sudo cat etc/shadow` — отображение шифров паролей пользователей с метаданными;  
`cat etc/group` — список всех групп;  
su *username* — сменить пользователя;  
`id` — показывает информацию об идентификаторах пользователя и его группах в системах Linux.  

Вот расшифровка вашего вывода:

- **`uid=1000(dufchik)`** — **User ID**. Уникальный числовой идентификатор вашего пользователя (`dufchik`). В большинстве дистрибутивов нумерация обычных пользователей начинается с `1000`.
    
- **`gid=1000(dufchik)`** — **Group ID**. Идентификатор вашей _основной (первичной)_ группы. По умолчанию создается группа с тем же именем, что и у пользователя.
    
- **`groups=...`** — **Дополнительные группы**, в которых состоит ваш пользователь, и их числовые ID. Они определяют ваши права в системе:
    
    - **`sudo` (27)** — дает право выполнять команды с правами суперпользователя (`root`).
        
    - **`adm` (4)** — позволяет читать большинство системных логов (`/var/log`).
        
    - **`lxd` (114)** — дает доступ к управлению контейнерами LXD.
        
    - **`cdrom` (24)**, **`plugdev` (46)** — дают доступ к монтированию и работе с внешними устройствами (CD-ROM, флешки).
        
    - **`users` (100)**, **`dip` (30)**, **`lpadmin` (111)** — стандартные группы для общих пользователей, настройки Dial-Up/VPN и управления принтерами;

`last` — отображает информацию о входах в систему, выходах и перезагрузках;  
`who` — отображение информации о тех кто сейчас вошёл в систему;  
`w` — отображение пользователей вошедших в систему и того что они делают;  
`sudo useradd -m *username*` — добавление нового пользователя (добавлять их может только пользователь, принадлежащий к группе 27(sudo));  
`sudo passwd *username*` — присвоить пользователю пароль;  
`sudo userdel (-r) *username*` — удалить пользователя (флаг -r стирает пользователя вместе с его домашней директорией);  
`sudo groupadd *groupname*` — создать новую группу;  
`sudo groupdel *groupname*` — удалить группу;  
`sudo usermod -aG *groupname* *username*` — добавить пользователя в группу;  
`sudo usermod -aG sudo *username*` — дать пользователю права администратора (команда добавит пользователя в группу 27(sudo));  
`sudo deluser *username* *groupname*` — удалить пользователя из группы;  
`sudo deluser *username* sudo` или `sudo gpasswd -d *username* sudo` — лишить пользователя прав администратора;  
`sudo chown *username* *objname*` — изменить владельца файла или директории;  
`sudo chgrp *groupname* *objname*` — изменить группу файла или директории;  
`sudo chmod *1*+-*2* *objname*`, где `*1*: u/g/o`, `*2*: r/w/x` — добавление или удаление прав владельца/группы/остальных на чтение/запись/исполнение файла или директории. Можно прописывать несколько прав сразу через запятую, например: `sudo chmod g+w, o-r *objname*`. А можно прописать `ugo/a(all)=r/w/x`, что сведёт выбранный параметр к одному значению во всех полях. Также можно прописывать права с помощью цифр для каждого поля отдельно, к примеру, комбинация 777 в `sudo chmod 777 *objname*` даст права rwx всем полям. Работает как двоичный код:
- `--- — 0`;
- `--x — 1`;
- `-w- — 2`;
- `-wx — 3`;
- `r-- — 4`;
- `r-x — 5`;
- `rw- — 6`;
- `rwx — 7`.
Ещё один важный момент. Для защиты объектов в директории от случайного удаления может понадобиться **sticky bit**. Он отображается как флаг `t` в конце кода с правами директории. Всё дело в том что даже если право на изменение (w) объекта будет, скажем, только у владельца, то все остальные всё равно смогут его стереть, если в директории в которой объект расположен не будет стоять флаг `t`, даже несмотря на отсутствие у них соответствующего права в коде этого объекта. Поставить или убрать этот флаг можно следующими командами:
`sudo chmod o+-t *dirname*` или `sudo chmod [0-1][0-7][0-7][0-7] *dirname*`

### Сети:

`ping *address*`  — считывание пинга сервера по указанному адресу (Ctrl + C — прервать запись, Ctrl + Z — остановить вывод пинга в терминал (запись продолжится в фоновом режиме) (команда fg — возобновить вывод));  
`ifconfig` — отображает и настраивает сетевые интерфейсы, а также их IP-адреса и параметры сетевой карты;  
`ip addr show` — современный аналог `ifconfig`, выводящий полную информацию о сетевых интерфейсах и их IPv4/IPv6 адресах;  
`route` — показывает и позволяет редактировать системную таблицу маршрутизации IP-пакетов;  
`host` — выполняет простые DNS-запросы для быстрого преобразования доменных имён в IP-адреса и наоборот;  
`dig` — утилита для получения подробных данных из DNS-серверов с детализацией записей всех типов;  
`netstat` — выводит активные сетевые подключения, открытые порты, статистику интерфейсов и таблицу маршрутизации;  

### Скриптинг:

#### Передача аргументов:

В случае `./*scriptname*.sh Vasya Petya Kolya`

`$0` равен `./*scriptname*.sh`;  
`$1` равен `Vasya`;  
`$2` равен `Petya`;  
`$3` равен `Kolya`;  

 ``*varname*=`*command*`/$(*command*)`` — записать результат выполнения команды в переменную;  
 `read -p "*comment*" *varname*` — сохранить ввод пользователя в переменную;
